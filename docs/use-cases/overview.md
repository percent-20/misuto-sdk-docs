# Use Cases Overview

The Misuto SDK enables location-aware applications across diverse industries. Each vault engine and sensor type maps to real-world business problems.

## Why Location-Aware Vaults?

Traditional apps treat location as a filter. Misuto treats it as a **key** — physical presence becomes a cryptographic requirement for accessing data, triggering actions, or joining conversations.

This creates entirely new product categories:

- **Access tied to place** — Documents only readable at the office
- **Actions tied to presence** — Webhooks fire when employees arrive at a job site
- **Secrets tied to proximity** — Files self-destruct after first open at a specific location
- **Communication tied to context** — Chat rooms that only exist at a physical venue

## Sensor Combinations

The power multiplies when you combine sensors:

| Combination | Use Case |
|-------------|----------|
| GPS + Time | Time-limited access at a location |
| GPS + BLE | Precise indoor positioning |
| GPS + WiFi | Network-verified location proof |
| GPS + NFC | Physical tap-to-access |
| GPS + Barometer | Floor-specific access in buildings |
| GPS + Camera | Object-verified location access |
| All + Shake | Emergency exit for classified scenarios |
